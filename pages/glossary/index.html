Glossary

Glossary with several platforms, tools, models and benchmarks. Just to be aware of the newest trends.

A

Active Learning: process of prioritising the data which needs to be labelled in order to have the highest impact to training a supervised model.

Airflow (Apache): platform to programmatically author, schedule and monitor workflows as directed acyclic graphs (DAGs) of tasks. The Airflow scheduler executes tasks on an array of workers while following the specified dependencies. The user interface makes it easy to visualize pipelines running in production, monitor progress, and troubleshoot issues when needed. Is scalable, dynamic and extensible.

AutoML (Google): technology that allows to train custom machine learning models with minimal effort and machine learning expertise. It covers the complete pipeline from the raw dataset to the deployable machine learning model.

B

Batch Jobs: jobs that can run without end user interaction, or can be scheduled to run as resources permit.

BERT: language representation model which stands for Bidirectional Encoder Representations from Transformers. See wiki post.

BLEU: stands for Bilingual Evaluation Understudy. Is a metric for evaluating the quality of machine translated text from one natural language to another.

C

CNN: stands for Convolutional Neural Network. Is a neural network that employs a convolution at least in one layer. Is commonly applied to computer vision.

CometML: platform that allows data scientists to track code, experiments, and results on ML projects. Itâ€™s fast, simple, and free for open source projects.

D

Dask: library for parallel computing in Python. Allows dynamic task scheduling and big data collection. It also integrates with common Data Science libraries such as Pandas, Numpy or Scikit-Learn.

DAWNBench: a Stanford-backed competition website for assessing things like inference cost, training cost, and training time for various AI tasks on different cloud infrastructures.

Docker: tool that allows to create, design and run applications by using containers. In turn, a Docker Container is a standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another.

E

ELECTRA: stands for Efficiently Learning an Encoder that Classifies Token Replacements Accurately. Is a language pre-training model that provides the benefits of BERT but learns far more efficiently.

G

GAN: stands for Generative Adversarial Network. Is an architecture that use two neural networks, set one against the other in order to generate synthetic instances of data. The goal is that the final data can pass for real data.

GLUE: stands for General Language Understanding Evaluation. Is a collection of NLU (Natural Language Understanding) tasks (e.g question answering or sentiment analysis) and an associated online platform for model evaluation, comparison and analysis. It favours models that can learn to represent linguistic knowledge in a way that facilitates sample-efficient learning and effective knowledge-transfer across tasks.

GPT-2: is a large transformer-based language model with 1.5 billion parameters. It is trained with the objective of predicting the next word, given all of the previous words within some text. The diversity of the dataset allows for containing naturally occurring demonstrations of many tasks across diverse domains.

Grafana: is an open-source analytics and monitoring solution for databases.

H

Hadoop (Apache): framework that allows for the distributed processing of large data sets across clusters of computers using simple programming models. It is designed to scale up from single servers to thousands of machines, each offering local computation and storage. Rather than rely on hardware to deliver high-availability, the library itself is designed to detect and handle failures at the application layer, so delivering a highly-available service on top of a cluster of computers, each of which may be prone to failures.

K

Keras: is a high-level neural networks API capable of running on top of , for example, TensorFlow. It was developed with a focus on enabling fast experimentation.

Kafka (Apache): distributed streaming platform used for building real-time data pipelines and streaming apps. It is horizontally scalable, fault-tolerant and wicked fast.

Kubeflow: make deployments of machine learning (ML) workflows on Kubernetes simple, portable and scalable. The goal is not to recreate other services, but to provide a straightforward way to deploy best-of-breed open-source systems for ML to diverse infrastructures.

Kubernetes: is an open-source system for automating deployment, scaling, and management of containerized applications.

L

LSTM: stands for Long Short-Term Memory. Is a recurrent neural network architecture. A common LSTM unit is composed of a cell, an input gate, an output gate and a forget gate. The cell remembers values over arbitrary time intervals and the three gates regulate the flow of information into and out of the cell.

M

Metaflow: Successful data science projects are delivered by data scientists who can build, improve, and operate end-to-end workflows independently, focusing more on data science, less on engineering.

MLPerf: fair and useful benchmarks for measuring training and inference performance of ML hardware, software, and services.

MLflow: open source platform to manage the ML lifecycle, including experimentation, reproducibility and deployment.

P

PCA: stands for Principal Component Analysis. Is a non-parametric method of extracting relevant information from confusing data set. Given a complex dataset, it reduces to a lower dimension revealing the sometimes hidden, simplified structure that often underlie it

PyTorch: is a machine learning framework that accelerates the path from research prototyping to production deployment.

R

RAPIDS: is a suite of software libraries and APIs that gives the ability to execute end-to-end data science and analytics pipelines entirely on GPUs. RAPIDS utilizes NVIDIA CUDA primitives for low-level compute optimization, and exposes GPU parallelism and high-bandwidth memory speed through user-friendly Python interfaces.

Reinforcement Learning: Machine Learning area that studies how an action maximizes the reward in a particular situation. It is employed by various software and machines to find the best possible behaviour or path. Is the third field of study alongside supervised and unsupervised learning.

RNN: stands for Recurrent Neural Networks. Is neural network architecture where direct nodes from direct nodes where connections between nodes form a directed graph along a temporal sequence.

S

SHAP: stands for SHapley Additive exPlanations and it is an interpretability metric as it explains the output of any machine learning model. It interprets the impact of having a certain value for a given feature in comparison to the prediction we’d make if that feature took some baseline value.

Spark (Apache): unified analytics engine for large-scale data processing.

T

Tensorflow: end-to-end open source machine learning platform to help developing and train ML models.

U

UMAP: stands for Uniform Manifold Approximation and Projection and it is a manifold learning technique for dimension reduction. UMAP is constructed from a theoretical framework based in Riemannian geometry and algebraic topology. Is scalable and has no computational restrictions on embedding dimension.
